# Memory vault index

Plain-markdown long-term memory. The assistant reads this at session start and appends as it learns. One file per topic; one line per file below.

This index ships empty. Your own memory files are gitignored, so what you add here stays on your machine.

- (example) `decisions.md` — durable decisions and why
- (example) `people.md` — recurring people, who they are
- (example) `active.md` — current state: what's in flight, open questions

Rule: keep this index under ~150 lines; split by topic if it grows. Don't store what the code or git history already records.
